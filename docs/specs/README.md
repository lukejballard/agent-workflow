Specs policy

This directory contains working and completed specifications for the portable
supermind package and its source-repo maintenance.

Policy
- Active implementation or cleanup specs belong under `docs/specs/active/`.
- Completed specs belong under `docs/specs/done/`.
- Research that materially affects a package decision belongs under `docs/specs/research/`.

Guidance
- Keep specs focused on the package boundary: `.github/` as product, `docs/` as maintainer layer.
- Prefer explicit requirements and acceptance criteria over aspirational roadmap language.
- Archive or remove specs that only describe deleted runtime, frontend, or evaluation systems.
