# SDK Reference

## Current State

This repository snapshot does not contain a verified SDK implementation or a checked-in
runtime service surface that would allow this page to document SDK behavior honestly.

## What This Means

The repo should not currently claim a stable SDK contract for:
- event capture
- background delivery
- transport guarantees
- authentication headers
- error-handling semantics

## Future SDK Documentation Requirements

A future SDK reference should only be published once the repository contains:
1. the actual SDK source
2. validated payload models
3. integration tests against the real service boundary
4. clear failure-mode and versioning guarantees

Until then, treat older SDK-oriented notes in archived materials as historical context,
not as active contract documentation.
