# Quickstart

## What You Can Do In This Snapshot

This repository is being simplified into a compact prompt/meta supermind
package. The practical maintainer quickstart is:

1. Read `AGENTS.md`.
2. Read `.github/AGENTS.md`.
3. Read `.github/agent-platform/workflow-manifest.json`.
4. Read `docs/product-direction.md` and `docs/architecture.md`.
5. For non-trivial work, create or refresh a working spec in `docs/specs/active/`.

## Core Surfaces

- `.github/` is the canonical package source.
- `docs/` is the maintainer documentation layer.

## Working Safely

When changing the package:
- prefer verified statements over reconstructed history
- keep historical material clearly marked as archival
- do not reintroduce runtime-product claims unless the package boundary changes
- keep `.github` contracts, docs, and specs aligned in the same change set
